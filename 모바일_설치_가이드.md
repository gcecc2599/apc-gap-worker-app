# 📱 APC GAP 기록관리 - 모바일 설치 가이드

## 설치 방법

### 방법 1: 웹 브라우저에서 직접 설치 (추천) ⭐

#### Android (안드로이드)
1. **Chrome 브라우저** 실행
2. 웹사이트 주소 입력: `http://서버주소:8787`
   - 예: `http://192.168.0.100:8787` (로컬 네트워크)
   - 예: `https://yourdomain.com` (온라인 배포 시)
3. 화면 오른쪽 상단 **⋮ (점 3개)** 메뉴 클릭
4. **"홈 화면에 추가"** 또는 **"앱 설치"** 선택
5. 설치 확인 팝업에서 **"설치"** 클릭
6. ✅ 완료! 홈 화면에서 앱 아이콘 확인

#### iOS (아이폰)
1. **Safari 브라우저** 실행 (Chrome 불가)
2. 웹사이트 주소 입력: `http://서버주소:8787`
3. 하단 중앙 **□↑ (공유)** 버튼 클릭
4. **"홈 화면에 추가"** 선택
5. 앱 이름 확인 후 **"추가"** 클릭
6. ✅ 완료! 홈 화면에서 앱 아이콘 확인

---

### 방법 2: 오프라인 설치 (PWA 단일 파일)

1. 이 ZIP 파일의 압축을 풉니다
2. `install.html` 파일을 모바일로 전송
3. 모바일에서 `install.html` 파일을 브라우저로 엽니다
4. 화면의 안내에 따라 홈 화면에 추가합니다

## 접속 정보

### 로컬 네트워크 (같은 Wi-Fi)
- 주소: `http://[PC의 IP주소]:8787`
- PC IP 주소 확인 방법:
  - Windows: 명령 프롬프트에서 `ipconfig` 입력
  - "IPv4 주소" 확인 (예: 192.168.0.100)

### 인터넷 배포
- 서버 배포 후 제공되는 주소 사용
- HTTPS 권장 (PWA 필수 사항)

## 주의사항

⚠️ **Android**: Chrome 브라우저 필수 (다른 브라우저는 PWA 설치 불가)
⚠️ **iOS**: Safari 브라우저 필수 (Chrome은 홈 화면 추가만 가능)
⚠️ **HTTPS 권장**: 로컬 네트워크(http://)도 작동하지만, 인터넷 배포 시 HTTPS 필수

## 문제 해결

### 앱 설치 옵션이 보이지 않음
- 브라우저를 최신 버전으로 업데이트
- manifest.json과 Service Worker가 정상 작동하는지 확인
- HTTPS 또는 localhost에서 접속 확인

### 앱이 설치되지 않음
- 브라우저 캐시 삭제 후 재시도
- 기기 재시작 후 재시도

### 앱 아이콘이 표시되지 않음
- `public/icon-192.png`와 `icon-512.png` 파일 확인
- 브라우저 개발자 도구에서 manifest.json 확인
